# 李群太｜8周测试开发 Offer 冲刺计划（长期记忆掌握版）

> 目标不是“每天学一个知识点”，而是把高频能力反复调用，直到它们变成可以在面试中随时取出的自己的东西。

## 一、核心方法：不是死记曲线，而是“掌握循环”

每个知识点都要经历 5 步：

1. **理解**：知道它解决什么问题。
2. **闭卷回忆**：不看资料，自己讲出来。
3. **动手应用**：写代码、SQL、命令或测试方案。
4. **换场景迁移**：换一道题、换一个业务场景。
5. **面试输出**：回答为什么、边界、取舍和异常情况。

只有完成第 4、5 步，才算进入长期可用状态。

---

## 二、自适应复习规则

不机械固定“第1、2、4、7天”，而是根据掌握情况决定下次复习：

| 本次表现 | 下次复习时间 |
|---|---|
| 完全想不起来 / 做错 | 次日 |
| 能说一半，但不稳定 | 2天后 |
| 基础题会，变式题不会 | 3到4天后 |
| 能独立完成并解释 | 7天后 |
| 能迁移并应对追问 | 14天后 |
| 连续两次稳定通过 | 28天后随机抽查 |

每次复习先闭卷，再看笔记。重读笔记不算复习。

---

## 三、每天的固定结构

### 工作日 50到70分钟

1. **旧知识调用 15到20分钟**
   - 从复习队列抽 2到3个旧知识
   - 口述、手写或做题

2. **主线训练 25到35分钟**
   - 当天只攻 1 个主问题
   - 不开太多新坑

3. **迁移输出 10到15分钟**
   - 放进自己的项目或真实业务案例
   - 或录一段 2到5分钟面试回答

4. **登记下次复习 2分钟**
   - 根据表现安排次日、2天后、4天后、7天后或14天后

### 极累保底版 20分钟

- 1道 Python / SQL / XPath
- 1个旧知识口述
- 1个项目问题回答

### 周末 2到3小时

- 30分钟：本周旧知识混合测试
- 60分钟：完整编码或自动化练习
- 30分钟：项目迁移
- 30到60分钟：模拟面试、复盘、投递

---

## 四、五条长期主线

整个8周只围绕这5条主线反复加深，不每天换地图。

### 主线A：Python
字符串、列表、字典、函数、异常、文件、日期、OOP、常见编码题。

### 主线B：SQL与数据库
JOIN、聚合、窗口函数、去重、Top N、事务、索引、锁、数据核对。

### 主线C：API与pytest
HTTP、鉴权、幂等、接口测试设计、requests、fixture、参数化、SQL断言、CI。

### 主线D：Selenium与UI自动化
XPath、CSS、等待、Page Object、稳定性、iframe、window、异常处理。

### 主线E：项目与面试
4个项目、4个真实案例、自我介绍、测试方案、问题定位、HR与薪资。

Appium、性能、安全、分布式作为“支线”，只有岗位需要时才加深。

## 七、最终掌握标准

一个知识点必须同时满足：

- 不看资料能讲清楚
- 能完成陌生题
- 能用于自己的项目或案例
- 能回答为什么、边界、替代方案
- 7到14天后仍能独立调用

达不到，就不是失败，只是说明它还没真正“归你”。重新进入复习队列即可。
